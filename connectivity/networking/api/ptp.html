Precision Time Protocol (PTP)

  • Supported features

  • API Reference

Overview

PTP is a network protocol implemented in the application layer, used to synchronize clocks in a computer network. It’s accurate up to less than a microsecond. The stack supports the protocol and procedures as defined in the IEEE 1588-2019 standard (IEEE Standard for a Precision Clock Synchronization Protocol for Networked Measurement and Control Systems). It has multiple profiles, and can be implemented on top of L2 (Ethernet) or L3 (UDP/IPv4 or UDP/IPv6). Its accuracy is achieved by using hardware timestamping of the protocol packets.

Zephyr’s implementation of PTP stack consist following items:

  • PTP stack thread that handles incoming messages and events

  • Integration with ptp_clock driver

  • PTP stack initialization executed during system init

The implementation automatically creates PTP Ports (each PTP Port corresponds to unique interface).

Enabling the stack

The following configuration option must me enabled in prj.conf file.

  • CONFIG_PTP

Testing

The stack has been informally tested using the Linux ptp4l daemons. The PTP sample application from the Zephyr source distribution can be used for testing.
