Detailed Description

IPC backend configuration structure.

This structure is used for configuration backend during registration.

◆ send_critical

int(* ipc_service_backend::send_critical) (const struct device *instance, void *token, const void *data, size_t len)

Pointer to the function that sends critical messages bypassing flow control.

This function sends high-priority messages directly, bypassing busy checks and normal queuing mechanisms. It should be used only for critical system notifications like crash reports or fatal errors.

Parameters
[in]instanceInstance pointer.
[in]tokenBackend-specific token.
[in]dataPointer to the buffer to send.
[in]lenNumber of bytes to send.
Return values
-EINVALwhen instance is invalid.
-ENOENTwhen the endpoint is not registered with the instance.
-EBADMSGwhen the message is invalid.
-ENOTSUPwhen critical send is not supported.
-ENOMEMwhen even critical buffers are unavailable.
bytesnumber of bytes sent.
othererrno codes depending on the implementation of the backend.
