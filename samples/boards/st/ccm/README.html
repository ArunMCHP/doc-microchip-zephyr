Core Coupled Memory (CCM)

Browse source code on GitHub

Overview

Show usage of the Core Coupled Memory (CCM) that is available on several STM32 devices. The very important difference with normal RAM is that CCM can not be used for DMA.

In this sample, the CCM is used to store data (i.e., as a DTCM). By prefixing a variable with __dtcm_data_section, __dtcm_bss_section or __dtcm_noinit_section, those variables are placed in the CCM.

The __dtcm_data_section prefix should be used for variables that are initialized. Like the normal data section the initial values take up space in the FLASH image.

The __dtcm_bss_section prefix should be used for variables that should be initialized to 0. Like the normal bss section they do not take up FLASH space.

The __dtcm_noinit_section prefix should be used for variables that don’t need to have a defined initial value (for example buffers that will receive data). Compared to bss or data the kernel does not need to initialize the noinit section making the startup slightly faster.

To add CCM support to a board, add the following line to the board’s DTS file chosen section:

zephyr,dtcm = &ccm0;

Building and Running

# From the root of the zephyr repository
west build -b None samples/boards/st/ccm
west flash

The first time the example is run after power on, the output will look like this:
