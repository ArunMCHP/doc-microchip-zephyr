◆ keep_section_info

bool llext_load_param::keep_section_info

Keep the ELF section data in memory after loading the extension.

This is needed to use some of the functions in ELF inspection APIs.

Note
Related memory must be freed by llext_free_inspection_data before the extension can be unloaded via llext_unload.

◆ pre_located

bool llext_load_param::pre_located

Use the virtual symbol addresses from the ELF, not addresses within the memory buffer, when calculating relocation targets.

It also means, that the application will take care to place the extension at those pre-defined addresses, so the LLEXT core doesn't have to do any allocation and copying internally. Any MMU permission adjustment will be done by the application too.
