microchip,pic32ck-sg-gc-pll
###########################

.. sidebar:: Overview

   :Name: ``microchip,pic32ck-sg-gc-pll``
   :Vendor: :ref:`Microchip Technology Inc. <dt_vendor_microchip>`
   :Used in: :zephyr:board-catalog:`List of boards <#compatibles=microchip,pic32ck-sg-gc-pll>` using
               this compatible

Description
***********



.. code-block:: none

   PIC32CK_SG_GC PLL clock

   Phase Locked Loop (pll), 12.7 MHz to 200 MHz output frequency or up to 1.6 GHz for
   fractional divider module use, from a 4 MHz to 48 MHz Reference Input Clock.

Child node properties
=====================

.. list-table::
   :widths: 1 1 4
   :header-rows: 1

   * - Name
     - Type
     - Details

   * - ``subsystem``
     - ``int``
     - .. code-block:: none

          Clock subsystem


       This property is **required**.

   * - ``pll-enable``
     - ``int``
     - .. code-block:: none

          Oscillator Enable
          0: to disable, 1: to enable



       Legal values: ``0``, ``1``

   * - ``pll-on-demand-en``
     - ``int``
     - .. code-block:: none

          0: The oscillator is always on
          1: The oscillator is running when a peripheral is requesting the oscillator to be used as a
          clock source. The oscillator is not running if no peripheral is requesting the clock source.
          Important: Initializing it with 1, along with clock enabled, can lead to indefinite wait
          for the clock to be on, if there is no peripheral request for the clock in the sequence of
          clock Initialization. If required, better to turn on the clock using API, instead of
          enabling both during startup.



       Legal values: ``0``, ``1``

   * - ``pll-bandwidth-sel``
     - ``string``
     - .. code-block:: none

          Select the pll0 closed loop filter bandwidth.

          Depending on the frequency after the reference divider FPFD. Selecting the correct filter
          bandwidth is important to operate the pll0'VCO in its best range.



       Default value: ``bwsel-0``

       Legal values: ``bwsel-0``, ``bwsel-1``, ``bwsel-2``, ``bwsel-3``, ``bwsel-4``, ``bwsel-5``, ``bwsel-6``, ``bwsel-7``

   * - ``pll-src``
     - ``string``
     - .. code-block:: none

          Reference source clock selection



       Default value: ``xosc``

       Legal values: ``gclk0``, ``gclk1``, ``gclk2``, ``gclk3``, ``gclk4``, ``gclk5``, ``gclk6``, ``gclk7``, ``gclk8``, ``gclk9``, ``gclk10``, ``gclk11``, ``xosc``, ``dfll48m``

   * - ``pll-feedback-divider-factor``
     - ``int``
     - .. code-block:: none

          This field determines the ratio of the pll0's VCO output frequency to the pll0 Reference
          input frequency.

          Writing to the FBDIV(feedback-divider-factor) bits will cause lock to be lost.
          The value of FBDIV, (i.e. pll0FBDIV) must be within the range 21 ≤ FBDIV ≤ 1023.
          The frequency of the Voltage Controlled Oscillator (VCO) giving the pll0 oscillation is
          given by the formula: FVCO = FCKR * (FBDIV / REFDIV), (i.e., Must be between 800 MHz and
          1600 MHz).
          fCKR(source clock), REFDIV(ref-division-factor) and FBDIV must be selected to satisfy this
          condition.



   * - ``pll-ref-division-factor``
     - ``int``
     - .. code-block:: none

          This field determines the division factor of the pll0 input reference frequency.

          Writing to the REFDIV bits will cause lock to be lost.
          REFDIV value must be in the range of 1 ≤ REFDIV ≤ 63.
          The frequency after the reference divider (FPFD) is given by the formula:
          FPFD = FCKR / REFDIV (i.e., FPFD must always be between 4 MHz to 48 MHz.



   * - ``pll-freq-division-factor-int``
     - ``int``
     - .. code-block:: none

          This field determines the ratio of the pll0's VCO output frequency to the pll0 Reference
          input frequency.



   * - ``pll-freq-division-factor-rem``
     - ``int``
     - .. code-block:: none

          This field determines the ratio of the pll0's VCO output frequency to the pll0 Reference
          input frequency.
