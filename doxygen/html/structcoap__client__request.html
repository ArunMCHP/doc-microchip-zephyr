◆ multicast_timeout_ms

uint32_t coap_client_request::multicast_timeout_ms

Multicast response timeout in milliseconds.

When > 0, indicates a multicast request that accepts multiple responses within the timeout period. After the timeout, a final callback with source=NULL signals completion, after which no further callbacks will be issued. Multicast requests are always non-confirmable (RFC 7252).

Attention
Available only when the following Kconfig option is enabled: CONFIG_COAP_CLIENT_MULTICAST.
